Part 2 — Can you switch from Firebase to another backend?
Honest answer: Switching right now would be painful, but not catastrophic. Here's exactly why.

What makes it hard today
Your service files (qr-service.ts, comment-service.ts, etc.) call Firebase SDK directly. They use:

doc(), collection(), getDocs(), setDoc(), updateDoc() — Firestore-specific
onSnapshot() — this is the hardest one. It's a real-time listener pattern that no SQL database has natively
AuthContext directly uses Firebase Auth classes — onIdTokenChanged, signInWithEmailAndPassword, GoogleAuthProvider
So if you want to switch to say Supabase, Neon (PostgreSQL), or Appwrite:

All 738 lines of qr-service.ts would need to be rewritten
All real-time onSnapshot listeners in useQrDetail would need to be replaced with Supabase Realtime or WebSocket polling
AuthContext would need a full rewrite
Your data structure (nested Firestore subcollections like qrCodes/{id}/comments/{id}/likes/{id}) would need to be flattened into SQL tables
What would make it easy to switch (the fix)
The professional way to handle this is a Repository Pattern — you create a thin interface layer between your hooks and the actual database:

lib/
  repositories/
    interfaces/
      IQrRepository.ts   ← defines what methods exist
      IAuthRepository.ts
    firebase/
      FirebaseQrRepository.ts  ← Firebase implementation
      FirebaseAuthRepository.ts
    supabase/             ← could add later
      SupabaseQrRepository.ts
Your hooks would call IQrRepository.getById(id) and never know if it's Firebase or Supabase underneath. Switching backends would then just mean swapping which implementation is injected.

Right now your app skips this layer — hooks call services, services call Firebase directly. That's 2 layers when you need 3.

The real-time problem is the hardest part
Firebase Firestore's onSnapshot gives you instant live updates for free. If you switch to PostgreSQL (Supabase, Neon, etc.) you'd need to either:

Use Supabase Realtime (similar concept, works well)
Use WebSockets from your Express server
Accept polling every few seconds (simpler but less live)
Supabase is actually the closest drop-in replacement — it has auth, real-time subscriptions, and PostgreSQL. The migration would still take significant work but the concepts map 1:1.
