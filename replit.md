# QR Guard

A full-stack mobile-first QR code scanning and management app built with Expo (React Native) and Express.

## Architecture

- **Frontend**: Expo (React Native) with Expo Router for navigation, running on port 8081
- **Backend**: Express server running on port 5000 (only handles `/api/qr/decode-image`)
- **Primary Database**: Firebase Firestore (all app data: QR codes, reports, comments, users, favorites, follows, feedback)
- **Realtime Database**: Firebase Realtime Database (live notifications for followers when QR codes get new comments/reports)
- **Auth**: Firebase Auth ‚Äî email/password + Google OAuth (`expo-auth-session`)
- **PostgreSQL / Drizzle**: Reserved for future migration at ~10k users (`server/storage.ts` ‚Äî not imported anywhere yet)

## Project Structure

- `app/` ‚Äî Expo Router screens
  - `(tabs)/index.tsx` ‚Äî Home screen with notification bell + RTDB-powered notification panel
  - `(tabs)/scanner.tsx` ‚Äî QR scanner (camera + image gallery)
  - `(tabs)/history.tsx` ‚Äî Scan history + favorites
  - `(tabs)/profile.tsx` ‚Äî Full profile: photo upload (camera/gallery), stats (following QRs, scans, comments, total likes), settings shortcut
  - `(tabs)/qr-generator.tsx` ‚Äî Branded QR (QR Guard logo + UUID + user info) or Private/No-trace mode, custom logo with 5 position options (center, top-left, top-right, bottom-left, bottom-right)
  - `(tabs)/settings.tsx` ‚Äî Account settings, feedback form, delete account
  - `(auth)/` ‚Äî Login + Register screens
  - `qr-detail/[id].tsx` ‚Äî QR detail with offline mode (cached content + "Enable internet" CTA), real-time Firestore listeners, skeleton loading, threaded comments, QR owner panel (followers list, inbox), messaging modal
- `server/` ‚Äî Express backend (`routes.ts` only does image decode via Jimp + jsQR)
- `shared/` ‚Äî Drizzle schema (future PostgreSQL)
- `components/` ‚Äî Reusable React Native components
- `contexts/AuthContext.tsx` ‚Äî Firebase Auth with `onIdTokenChanged` for automatic token refresh
- `lib/firebase.ts` ‚Äî Firebase app, Auth, Firestore, Realtime Database
- `lib/firestore-service.ts` ‚Äî All Firebase/Firestore/RTDB operations
- `lib/query-client.ts` ‚Äî TanStack Query + API request utility
- `assets/` ‚Äî Static assets

## Firebase Integration (complete)

### Firestore Collections
- `qrCodes/{qrId}` ‚Äî QR code data (content, contentType, scanCount, commentCount)
- `qrCodes/{qrId}/reports/{userId}` ‚Äî Reports (safe/scam/fake/spam) per user
- `qrCodes/{qrId}/followers/{userId}` ‚Äî Followers per QR code
- `qrCodes/{qrId}/comments/{commentId}` ‚Äî Comments (soft-delete supported)
- `qrCodes/{qrId}/comments/{commentId}/likes/{userId}` ‚Äî Like/dislike per comment per user
- `qrCodes/{qrId}/comments/{commentId}/reports/{userId}` ‚Äî Comment reports
- `users/{userId}` ‚Äî User profile (soft-delete support)
- `users/{userId}/scans/` ‚Äî Scan history
- `users/{userId}/favorites/` ‚Äî Favorited QR codes
- `users/{userId}/following/` ‚Äî Followed QR codes
- `users/{userId}/comments/` ‚Äî User's comments (for profile)
- `users/{userId}/generatedQrs/` ‚Äî Branded QR codes generated by user (content, contentType, uuid, qrCodeId, createdAt)
- `qrMessages/{messageId}` ‚Äî Private messages sent to branded QR code owners (fromUserId, fromDisplayName, toUserId, qrCodeId, qrBrandedUuid, message, read, createdAt)
- `feedback/` ‚Äî Submitted feedback
- `users/{userId}.photoURL` ‚Äî Profile photo stored as base64 data URI
- QR ownership: when `saveGeneratedQr` is called with `branded=true`, the `qrCodes/{qrId}` document is updated with `ownerId`, `ownerName`, `brandedUuid`, `isBranded` (first creator claims ownership)
- `users/{userId}.totalLikesReceived` ‚Äî Incremented when user's comments are liked

### Real-time Subscriptions (onSnapshot)
- `subscribeToQrStats` ‚Äî Live scan count + comment count on QR detail
- `subscribeToQrReports` ‚Äî Live report counts ‚Üí trust score recalculation
- `subscribeToComments` ‚Äî Live first-page comments
- `getCommentUserLikes` ‚Äî Batch-fetches current user's like status for visible comments

### Firebase Realtime Database
Path: `notifications/{userId}/items/{notifId}`
- `subscribeToNotificationCount` ‚Äî Unread count for tab badge + notification bell
- `subscribeToNotifications` ‚Äî Full notification list for panel
- `markAllNotificationsRead` ‚Äî Bulk mark-read via atomic update
- `clearAllNotifications` ‚Äî Delete all notifications for user
- `notifyQrFollowers` ‚Äî Writes notifications to all followers when a comment/report is added

QR code IDs are SHA-256 hashes of the QR content (first 20 characters), not UUIDs.

## Environment Variables

- `PORT` ‚Äî Server port (defaults to 5000)
- `REPLIT_DEV_DOMAIN` ‚Äî Used for CORS and Expo proxy URL
- `EXPO_PUBLIC_DOMAIN` ‚Äî Backend API URL
- Firebase keys stored in `.replit` `[userenv.shared]` and hardcoded as fallbacks in `lib/firebase.ts`

## Important Notes

- `server/storage.ts` is **intentionally not imported** anywhere. It is the Drizzle/PostgreSQL migration layer to activate at ~10k users. Do NOT delete it.
- No backend auth routes ‚Äî all auth is client-side Firebase. The Express server only decodes QR images from uploaded photos.
- Firestore security rules should enforce read/write permissions by `request.auth.uid`.
